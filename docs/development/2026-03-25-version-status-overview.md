# AI 人才协作平台版本与状态总览（能力现状、Fallback/Mock 与上线前提）

更新时间：2026-03-25

这份文档只回答一个问题：项目现在到底做到哪一步，哪些能力已经能演示，哪些还只是骨架，哪些还不能算正式上线能力。

## 1. 状态判定规则

为了避免“页面能点开”和“后端能上线”混为一谈，这里统一使用 4 种状态：

- `正式上线可测`
  - 已具备正式后端承载、正式鉴权、正式存储或正式回写链路
- `部分正式化`
  - 已有真实持久化或真实读模型，但只覆盖局部子链路，仍有 fallback 或上线保障缺口
- `演示可测`
  - 页面和流程可跑通，适合联调、演示、体验验证，但底层仍有 mock / fallback / 本地持久化依赖
- `骨架已就位`
  - 工程结构、实体、控制器或接口预留已存在，但还不能独立承载真实业务
- `待建设`
  - 尚未形成可联调能力或正式能力

## 2. 一句话结论

当前项目最准确的状态是：

- **前台、后台、root mock API 已达到“演示可测”。**
- **`spring-app` 已达到“部分正式化 + 局部持久化读写已完成”。**
- **整个项目还没有达到“正式上线可测”。**

## 3. 当前版本定位

### 3.1 当前适合什么

- 产品流程演示
- 前后端联调
- 小范围多角色测试
- 交互优化和页面打磨

### 3.2 当前不适合什么

- 直接承载公网真实用户
- 高并发正式业务写入
- 正式聊天回写与正式文件存储
- 依赖完整审计、监控、告警的生产环境

## 6. 最容易失真的口径

后续维护文档时，这几件事最容易被写偏：

- `root mock API 可运行` 不等于 `后端可上线`
- `有登录页` 不等于 `有正式认证鉴权`
- `页面能上传附件` 不等于 `有正式文件服务`
- `接了腾讯 IM SDK` 不等于 `聊天主链正式完成`
- `能调到 DeepSeek` 不等于 `全链路都是真实 AI`
- `协作空间能看见数据` 不等于 `数据已正式落库`
- `spring-app 有实体和控制器` 不等于 `主业务链已切正式服务`

## 7. 待正式化清单

当前最应该继续补齐的不是页面展示，而是工程闭环：

- 正式认证鉴权
- 任务主写链正式迁移到 `spring-app + MySQL`
- 腾讯 IM 回调、回写、验签、重试与幂等
- 正式文件上传与对象存储
- 正式实时状态同步
- 监控、告警、限流、压测
