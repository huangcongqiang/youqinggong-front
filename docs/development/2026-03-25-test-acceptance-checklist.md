# AI 人才协作平台测试清单与验收清单

更新时间：2026-03-25

这份文档用于联调、提测、上线前回归时统一检查项目是否可用，并明确哪些能力当前只能算“演示可测”。

## 1. 文档目的与适用范围

适用场景：

- 日常联调
- 提测前自查
- 上线前回归
- 交付给其他开发或测试同学

覆盖范围：

- 官网
- 企业端
- 人才端
- 聊天
- 协作空间
- 后台管理
- 后端主链
- 腾讯 IM
- AI 拆解
- 部署前回归

不覆盖：

- 未来规划
- 未落地的正式高并发能力

## 2. 环境与版本说明

测试前至少确认：

- 当前前台地址可用
- 当前后台地址可用
- 当前后端地址可用
- 腾讯 IM 是否已接真实配置
- DeepSeek 是否已接真实配置
- 当前使用的是 root mock backend 还是 `spring-app`

推荐同时准备：

- 企业端测试账号
- 人才端测试账号
- 管理后台账号

## 3. 验收状态标记规则

每项用例统一按下面 3 种状态记录：

- `正式上线可测`
  - 代表可用正式后端链路验收
- `演示可测`
  - 代表页面和流程可跑，但底层仍依赖 mock / fallback / 本地状态
- `暂不可测`
  - 代表当前还没有形成有效测试能力

## 4. 官网测试清单

当前建议状态：`演示可测`

通过条件：

- 首页能稳定打开
- 首页是单列官网展示结构
- 企业案例轮播和人才案例轮播可正常切换
- 企业端 / 人才端入口分流正确
- 登录弹窗可打开
- 注册入口可进入独立注册页
- 联系方式和平台信息可见

## 5. 企业端测试清单

当前建议状态：`演示可测`

### 5.1 注册与入驻

通过条件：

- 企业端可注册
- 登录后能进入企业端
- 入驻页为单列步骤式
- 合作偏好为结构化选项
- 资料可直接上传或稍后补交
- 提交后进入企业工作台

### 5.2 发布任务

通过条件：

- 能输入任务标题、需求描述、预算
- 不选模板也能继续
- AI 拆解能正常生成
- AI 标签更偏业务 / 交付标签
- 推荐工具优先国内可用工具
- 发布成功后自动弹出推荐人才

### 5.3 选人与聊天

通过条件：

- 推荐人才可直接选择
- 也可跳过后续再选
- 选中人才后自动进入正确聊天房间
- 聊天列表主标题显示任务名称
- 最近更新的房间排最前

### 5.4 任务确认与变更

通过条件：

- 企业可发送任务确认卡
- 人才未处理前，企业不能继续发送第二个变更
- 企业可以撤回当前待确认变更
- 企业可修改说明、工期、预算后重新发送
- 历史显示版本和处理记录

### 5.5 协作与验收

通过条件：

- 企业协作空间可在多个任务之间切换
- 企业可查看节点、人才提交、AI 审核、企业反馈
- 企业可发起提前完成
- 企业可发起取消任务
- 企业可完成验收并打 `S / A / B` 评级

## 6. 人才端测试清单

当前建议状态：`演示可测`

### 6.1 注册与资料

通过条件：

- 人才端可注册
- 登录后能进入人才端
- 可完善资料、专长、作品、档期

### 6.2 任务与接单

通过条件：

- 人才端能查看任务广场
- 被企业选中后工作台能看到醒目提醒
- 被选中后聊天页可看到任务确认卡

### 6.3 任务确认与协作

通过条件：

- 人才可确认任务
- 人才可提出修改意见
- 企业更新后，人才能看到新的版本
- 人才可在协作空间提交进度、需要协助事项和附件
- 协作空间能查看任务详情弹窗

### 6.4 提前完成 / 取消任务 / 评分

通过条件：

- 人才能看到企业发起的提前完成请求
- 可同意提前完成或继续执行
- 可同意取消或拒绝取消
- 验收完成后可给企业评分

## 7. 聊天测试清单

当前建议状态：`演示可测`

通过条件：

- 会话列表能正常显示任务名称、协作对象和阶段
- 会话列表位于顶部，并支持横向滚动
- 长标题和最后一条消息使用省略号
- 聊天内容区为单列、固定高度、内部滚动
- 系统消息居中显示
- 对方消息居左
- 自己消息居右
- 文本消息可发送
- 图片消息可发送
- 文件消息可发送
- 任务确认卡作为消息流的一部分显示
- 查看任务详情弹窗可打开
- 沟通纪要通过悬浮入口 + 弹窗查看

## 8. 协作空间测试清单

当前建议状态：`演示可测`

### 8.1 企业端协作空间

通过条件：

- 支持多任务切换
- 节点按任务模块或工作日组织
- 点击节点能查看交付、AI 审核、企业反馈和 AI 补充解释
- 可发起提前完成
- 可发起取消任务

### 8.2 人才端协作空间

通过条件：

- 保持单列执行台
- 能看到当前接的任务
- 能提交完成度、进展说明、需要协助事项和附件
- 成功提交后页面即时回显新记录

## 9. 后台管理测试清单

当前建议状态：`演示可测`

通过条件：

- 管理后台可登录
- 列表主视图正常
- 用户审核可执行
- 任务流转可执行
- 风控工单处理可执行
- 抽屉 / 操作栏交互正常

## 10. 后端主链测试清单

当前建议状态：

- root mock backend：`演示可测`
- `spring-app`：`部分正式化中`

通过条件：

- 登录 / 注册 / `me` 正常
- 企业入驻、人才入驻正常
- 发布任务正常
- AI 拆解正常
- 推荐人才正常
- 选人后可创建或定位会话
- 任务确认 / 修改 / 撤回正常
- 进度提交正常
- 提前完成闭环正常
- 取消任务闭环正常
- 验收与评分正常
- 评分能回流到首页和工作台摘要

## 11. 腾讯 IM 测试清单

当前建议状态：`演示可测`

通过条件：

- 配置下发正常
- 用户身份映射正常
- 房间绑定正常
- 文本消息正常
- 图片 / 文件消息正常
- 发送失败不影响已同步聊天记录展示
- 当前房间不误跳到历史房间

当前仍应标注为演示可测的原因：

- 回调持久化、异常治理、幂等、生产监控还未完全正式化

## 12. AI 拆解测试清单

当前建议状态：`演示可测`

通过条件：

- 需求输入后能得到结构化拆解
- 预算、工期、标签、推荐人才有返回
- 可基于用户修改后重新生成
- 标签不再默认输出技术框架
- 工具建议优先推荐国内可用工具

## 13. 部署前回归清单

当前建议状态：`暂不可测` 到 `演示可测` 之间，取决于是否已切正式服务

通过条件：

- 环境变量齐全
- 域名与 HTTPS 配置正确
- 构建产物可用
- 数据库初始化和迁移可执行
- 腾讯 IM 配置正确
- DeepSeek 配置正确
- 上传能力可用
- 健康检查可访问
- 日志与告警路径明确
- 回滚方案明确

## 14. 当前应明确标为“演示可测”的能力

这些能力当前不建议标成“正式上线可测”：

- root mock backend 驱动的整条业务流程
- 当前轻量登录态与本地持久化会话
- 聊天页的大部分业务状态同步
- 协作空间节点图与节点弹窗
- 提前完成、取消任务、评级结算整条业务流
- 首页案例轮播与评分回流展示
- 企业 / 人才工作台统计摘要
- 后台管理当前的数据流转动作
- DeepSeek 与 fallback 混合能力
- 腾讯 IM 当前集成整体状态
- 当前文件上传体验与附件索引
- root mock backend 的并发承载能力

## 15. 当前应明确标为“正式上线前仍待补齐”的能力

- `spring-app` 正式认证与权限
- 任务主写链正式迁移
- 腾讯 IM 回调持久化与异常治理
- 正式文件上传服务
- 正式实时状态同步
- 监控、告警、限流、压测

## 16. 使用建议

这份文档建议在 3 个场景里使用：

1. 每次大功能联调前，先确认本轮要测哪些模块
2. 每次提测前，按角色走一遍关键主流程
3. 上线前，把“演示可测”与“正式上线可测”严格区分开
