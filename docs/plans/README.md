# plans 目录说明

这个目录回答：项目分阶段要怎么推进。

补充说明：

- 这里更适合放“阶段方案基线”和“历史快照”
- 如果内容已经在 `requirements / design / technical / development` 里持续维护，这里不再重复维护成第二份事实源
- 所以 `plans/` 里的文档，默认不是当前实现的唯一依据

适合谁看：

- 产品负责人
- 项目经理
- 技术负责人

建议先看：

1. [总体设计 / MVP 计划](./2026-03-21-ai-talent-market-design.md)
2. [H5 / PC 技术蓝图](./2026-03-28-h5-pc-technical-blueprint.md)

这个目录里应该放什么：

- 阶段目标
- MVP 范围
- 里程碑
- 后续路线
- 历史方案快照
- 目标态蓝图，例如 H5 / PC 技术方案蓝图

这个目录里不应该放什么：

- 详细接口字段
- 数据库实体定义
- 服务器配置
- 当前实现状态的逐日更新
