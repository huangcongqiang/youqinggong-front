# technical 目录说明

这个目录回答：系统是怎么实现的。

适合谁看：

- 前端开发
- 后端开发
- 测试
- 需要确认集成边界的第三方同学

建议阅读顺序：

1. [技术文档](./2026-03-21-ai-talent-market-technical.md)
2. [API 详细契约文档](./2026-03-21-api-contract.md)  
   这份文档以接口契约、字段口径、请求响应结构和错误码为主。
3. [Spring App 结构说明](./2026-03-21-spring-app-structure.md)
4. [腾讯 IM 集成说明](./2026-03-21-tencent-im-integration.md)
5. [目标态技术蓝图](../plans/2026-03-28-h5-pc-technical-blueprint.md)

这个目录里应该放什么：

- 当前实现与已定案的总体技术架构
- 前后端模块边界
- 端 / API / 核心服务的分层关系
- 数据流、实时链路和服务端兜底边界
- 接口契约
- 第三方集成方案

这个目录里不应该放什么：

- 需求优先级讨论
- 视觉稿说明
- 运维发布步骤
- 尚未定案的目标态蓝图
