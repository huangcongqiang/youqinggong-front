# technical 目录说明

这个目录回答：系统是怎么实现的。

适合谁看：

- 前端开发
- 后端开发
- 测试
- 需要确认集成边界的第三方同学

建议阅读顺序：

1. [技术文档](./2026-03-21-ai-talent-market-technical.md)
2. [API 详细契约文档](./2026-03-21-api-contract.md)

3. [Spring App 结构说明](./2026-03-21-spring-app-structure.md)
4. [腾讯 IM 集成说明](./2026-03-21-tencent-im-integration.md)

这个目录里应该放什么：

- 总体技术架构
- 前后端模块边界
- 数据流和服务分层
- 接口契约
- 第三方集成方案

这个目录里不应该放什么：

- 需求优先级讨论
