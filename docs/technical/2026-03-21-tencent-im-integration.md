# 腾讯 IM 集成说明

## 1. 目标

平台消息能力在 MVP 阶段先用 mock 数据占位，后续正式接入腾讯 IM，承担以下职责：

- 单聊
- 任务群聊
- 历史消息回溯
- 离线推送
- 系统消息触达

## 2. 为什么用腾讯 IM

- 即时消息基础能力成熟
- 会话、群组、离线推送能力完善
- 适合企业端与人才端围绕任务建立房间式沟通

## 3. 推荐接入边界

### 腾讯 IM 负责

- 会话建立
- 单聊 / 群聊
- 历史消息同步
- 推送与在线状态

### 平台负责

- 任务主状态
- 里程碑
- 附件索引
- AI 审查结果
- 验收节点
- 评分与信用画像

## 4. 推荐业务流程

1. 企业端确认选择人才
2. 平台创建任务协作记录
3. 平台调用腾讯 IM 创建任务房间
4. 将 `task_id` 与 `im_room_id` 建立映射
5. 任务消息通过腾讯 IM 传递
6. 系统关键摘要回写平台数据库

## 5. 数据表映射

已预留表：

- `chat_conversations`
- `chat_messages`
- `task_im_rooms`

推荐关系：

- 一个任务可绑定一个主任务房间
- 系统消息可双写到平台和 IM
- 附件元数据保留在平台数据库中

## 6. 接入建议

### 第一阶段

- 打通账号映射
- 创建任务房间
- 打通单聊 / 群聊
- 展示会话列表
- 建立 `/api/tencent-im/callback/events` 与 `/api/tencent-im/callback/messages` 回调入口

### 第二阶段

- 历史消息同步
- 系统消息模板
- 离线推送
- 会话摘要同步回任务详情

### 第三阶段

- AI 自动总结消息上下文
- 风险提醒
- 未读消息驱动任务运营提醒

## 7. 风险与注意事项

- 不要把任务状态完全依赖在 IM 消息里
- 不要把验收结论只留在聊天记录里
- 敏感资料仍应走平台自己的权限和存储体系
- 腾讯 IM 的会话能力应服务协作，不替代业务主数据
