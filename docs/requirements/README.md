# requirements 目录说明

这个目录只回答一件事：产品到底要做什么。

适合谁看：

- 产品经理
- 业务同学
- 第一次接手项目的人
- 需要确认范围边界的研发同学

建议先看：

1. [需求文档](./2026-03-28-enterprise-employment-collaboration-requirements.md)
2. [功能清单](./2026-03-30-product-feature-inventory.md)

这个目录里应该放什么：

- 功能范围
- 角色定义
- 核心业务链路
- 面向业务的功能清单
- MVP 边界
- 非功能性约束

这个目录里不应该放什么：

- 页面视觉细节
- 技术实现细节
- 部署说明
