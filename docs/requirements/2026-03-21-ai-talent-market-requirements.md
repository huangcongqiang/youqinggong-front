# AI 人才协作市场需求文档

## 1. 文档目的

沉淀平台的业务目标、角色需求、功能范围、核心流程和验收标准，作为后续产品迭代、设计评审、技术开发和项目验收的统一依据。

## 2. 项目定位

打造一款围绕 AI 交付协作场景的人才市场平台，连接企业端招聘方与人才端服务方，解决需求不清晰、沟通成本高、交付不可视、验收不透明的问题。

平台目标：

- 让招聘方可以更快发布任务并找到合适人才
- 让人才可以更好展示作品、管理档期并高效接单
- 让 AI 参与任务拆解、进度审查、验收提醒等关键节点
- 让后台可以统一审核、管理和风控

## 3. 用户角色

### 企业端招聘方

- 企业用户：上传企业证明材料后入驻
- 个人用户：申请虚拟企业并上传身份证等实名材料后入驻

核心诉求：

- 快速发布任务
- 自动拆解需求并评估工期
- 快速找到合适人才
- 在线沟通、跟进、验收

### 人才端服务方

核心诉求：

- 展示个人资料、专长、作品和评价
- 主动浏览任务或等待平台分配
- 管理接单日历和进行中任务
- 上传进度、和客户沟通、收获评价

### 平台管理员

核心诉求：

- 审核入驻资料
- 管理用户、任务、进度、评价和风险体系
- 处理异常任务和风控问题
- 手工创建用户与任务

## 4. 核心业务流程

### 企业端流程

1. 从官网首页进入企业端入口
2. 申请入驻
3. 上传企业资料或个人实名资料
4. 后台审核通过
5. 发布任务（语音 / 文字）
6. AI 拆解需求、工期、模块、风险
7. 企业端确认 AI 输出
8. 进入人才广场查看和选择人才
9. 双方确认需求与工期
10. 人才进入执行
11. AI 巡检进度并给出建议
12. 完工后提醒验收
13. 验收完成并评分

### 人才端流程

1. 从官网首页进入人才端入口
2. 提交人才入驻申请与实名资料
3. 填写资料、专长、作品、档期
4. 浏览任务广场或接受分配
5. 与企业端确认需求与工期
6. 在接单日期内持续上传进展
7. 通过消息列表与企业端沟通
8. 完工后等待验收
9. 对企业端进行评价
10. 在我的页面查看收入和作品积累

## 5. 功能需求

### 5.1 前台 H5 / PC

#### 首页

- 作为官网首页使用
- 展示平台定位、价值主张和流程说明
- 提供企业端与人才端入口
- 展示 AI 能力、平台案例和联系方式

#### 企业端工作台

- 入驻与审核状态查看
- 发布任务
- 语音或文字输入需求
- AI 拆解需求与工期
- 手动确认拆解结果
- 浏览人才广场
- 查看人才详情
- 和人才沟通并确认需求、工期
- 查看项目进度与验收状态

#### 人才端工作台

- 申请入驻
- 编辑个人资料、专长、作品、评价展示
- 浏览任务广场
- 查看自动分配任务
- 任务管理
- 接单日历
- 上传进度附件
- 消息中心
- 收入与作品统计

#### 协作空间

- 任务需求确认
- 工期协商
- 里程碑管理
- 进度上传
- AI 进度审查与修改建议
- 验收提醒
- 双向评分

### 5.2 后台管理

- 经营看板
- 用户管理
- 企业资料审核
- 个人实名与虚拟企业审核
- 任务管理
- 风控与异常提醒
- 手工创建用户
- 手工创建任务
- 页面主视图以列表和表格为主
- 操作入口统一放在表格右侧操作栏
- 创建、审核、流转、风控处理通过右侧抽屉完成
- 仅读取必要的协作审计留痕，不承接企业端 / 人才端 IM 主会话

### 5.3 AI 能力

- 任务文本 / 语音需求拆解
- 工期评估
- 技能标签提取
- 风险提醒
- 进度完整度检查
- 验收前提醒

### 5.4 消息能力

- 平台任务会话
- 任务系统消息
- 主要服务于企业端招聘方、人才端服务方和 AI 系统助手之间的协作
- 后台仅在投诉、争议、风控时查看摘要或审计留痕
- 后续接入腾讯 IM 的单聊 / 群聊 / 离线推送能力

## 6. 非功能需求

- 前台需支持 H5 与 PC 自适应
- 管理后台以 PC 宽屏为主
- 管理后台主色调采用蓝色、深蓝和白色
- 资料、身份证等敏感信息必须仅后台可见
- 核心页面结构需清晰，支持后续真实接口接入
- 代码结构需支持继续迭代

## 7. MVP 边界

本期优先实现：

- 页面骨架
- 角色流程展示
- Java mock API
- SQL 表结构
- AI 与腾讯 IM 接入预留

本期暂不实现：

- 真实登录注册
- 真实文件存储
- 真实支付与结算
- 真实大模型调用
- 真实腾讯 IM SDK 接入

## 8. 验收标准

### 产品层

- 能完整展示官网首页、企业端、人才端、后台管理四类页面层次
- 能演示企业端发布任务 -> AI 拆解 -> 匹配 -> 协作 -> 验收流程
- 能演示人才端看任务 -> 接单 -> 协作 -> 验收 -> 评价流程
- 能体现 H5 / PC 适配能力

### 技术层

- `frontend` 可运行
- `admin` 可运行
- `backend` 可启动并返回 JSON
- SQL 表结构覆盖核心业务实体
