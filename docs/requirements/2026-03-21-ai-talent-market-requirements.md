# AI 人才协作市场需求文档

## 1. 文档目的

沉淀平台的业务目标、角色需求、功能范围、核心流程和验收标准，作为后续产品迭代、设计评审、技术开发和项目验收的统一依据。

## 2. 项目定位

打造一款围绕 AI 交付协作场景的人才市场平台，连接企业端招聘方与人才端服务方，解决需求不清晰、沟通成本高、交付不可视、验收不透明的问题。

平台目标：

- 让招聘方可以更快发布任务并找到合适人才
- 让人才可以更好展示作品、管理档期并高效接单
- 让 AI 参与任务拆解、进度审查、验收提醒等关键节点
- 让后台可以统一审核、管理和风控

## 3. 用户角色

### 企业端招聘方

- 企业用户：上传企业证明材料后入驻
- 个人用户：申请虚拟企业并上传身份证等实名材料后入驻

核心诉求：

- 快速发布任务
- 自动拆解需求并评估工期
- 快速找到合适人才
- 在线沟通、跟进、验收

### 人才端服务方

核心诉求：

- 展示个人资料、专长、作品和评价
- 主动浏览任务或等待平台分配
- 管理接单日历和进行中任务
- 上传进度、和客户沟通、收获评价

### 平台管理员

核心诉求：

- 审核入驻资料
- 管理用户、任务、进度、评价和风险体系
- 处理异常任务和风控问题
- 手工创建用户与任务

## 4. 核心业务流程

### 企业端流程

1. 从官网首页选择企业端注册或登录
2. 新用户进入单列步骤式注册页，老用户通过登录弹窗进入
3. 上传企业资料或个人实名资料
4. 后台审核通过
5. 发布任务（语音 / 文字）
6. AI 拆解需求、工期、模块、风险
7. 企业端确认 AI 输出
8. 任务正式发布成功后，系统弹出 AI 推荐人才名单
9. 发布任务时需要填写预算，供人才端判断收入与接单意愿；若未填写，用户侧统一显示为“未填写预算”
10. AI 基于任务标签、档期、评分、作品与历史交付表现，从数据库先筛出 4 位候选人才并给出推荐理由
11. 企业端可直接从推荐弹窗中选择人才，或点击跳过，后续再主动进入人才广场做手动补充筛选
12. 选中人才后自动进入独立聊天页，并向人才端同步醒目的“待确认任务”提醒
13. 聊天页展示任务确认卡，并提供“查看任务详情”弹窗；任务详情、聊天确认卡和协作空间都要能看到当前预算、确认工期、协作安排和人才近期档期
14. 人才可直接确认任务，或提出修改意见；人才提出修改后，AI 先生成修改建议，企业再基于当前内容编辑并重新发送新版本的任务说明、工期、预算、协作安排与附加说明
15. 企业修改时要持续看到 AI 建议；企业重新发送前，AI 还需要基于最新版本再做一次复核，告诉企业“是否已经基本响应人才反馈”；如果是企业主动补充信息，AI 也要评估这次补充是否合理
16. 任务确认卡需要显示当前版本和最近版本记录；版本记录按 `第 1 版 -> 第 2 版` 顺序展示，旧版本状态更新为“已修改”，并记录本轮改了什么
17. 若企业端已发出新的变更确认单，在人才未处理前，不允许继续发送第二个；企业可先撤回本次变更，再重新编辑
18. 企业修改并重新发送后，人才端聊天列表、人才工作台和待确认入口都要出现新的醒目提醒
19. 双方确认需求与工期后进入执行阶段
20. AI 巡检进度并给出建议
21. 企业可在执行过程中发起“提前完成任务”，由 AI 先审核当前进度、质量和附件完整度
22. AI 审核通过后，由人才决定是否同意提前完成；若人才不同意，任务继续执行
23. 若人才同意提前完成，则由企业给出 S / A / B 交付评级，分别对应 100% / 80% / 30% 结算比例
24. 企业与人才任一方都可发起取消任务，但必须由对方明确同意后才生效
25. 完工后提醒验收
26. 人才继续对企业进行评分，沉淀企业合作信用

### 人才端流程

1. 从官网首页选择人才端注册或登录
2. 新用户进入单列步骤式注册页，老用户通过登录弹窗进入
3. 填写资料、专长、作品、档期
4. 浏览任务广场或接受分配
5. 被企业选中后，在聊天页收到任务确认卡和醒目提醒
6. 在聊天页查看当前确认版本、确认工期、协作安排和人才档期摘要，再决定确认或提出修改
7. 企业端更新后，人才端工作台与聊天列表都会同步出现第 N 版待确认提醒，再次查看新的任务确认版本并决定是否确认
8. 在接单日期内持续上传进展
9. 通过独立聊天页与企业端沟通
10. 当企业发起提前完成时，先查看 AI 审核意见，再决定同意或继续执行
11. 人才或企业都可发起取消任务，但取消需由对方确认
12. 完工后等待验收
13. 对企业端进行评价
14. 在我的页面查看收入、评级结果和作品积累

## 5. 功能需求

### 5.1 前台 H5 / PC

#### 首页

- 作为官网首页使用
- 展示精简的平台定位与价值主张
- 提供企业端与人才端入口
- 未登录时首页只展示注册入口和登录弹窗入口，不混放步骤式注册表单
- 展示 2 至 3 组核心差异点、平台案例和联系方式
- 首页整体保持简约，不堆叠过长流程说明、重复指标或测试性文案
- 首页视觉优先采用官网展示页结构，不使用工作台式统计堆叠
- 首页优先使用单列展示结构，让用户沿一个阅读路径往下浏览
- 案例区优先采用图片加文字的轮播展示方式，并区分企业案例与人才案例两组

#### 注册页

- 独立于官网首页
- 采用单列步骤式结构
- 第一步确认角色身份
- 第二步填写基础账号信息
- 第三步确认信息并创建账号
- 注册完成后自动进入对应角色的入驻流程

#### 登录

- 首页和导航栏点击登录时使用弹窗
- 不与注册表单混放在同一页
- 登录成功后直接进入当前账号对应业务端

#### 企业端工作台

- 以单列总览页展示企业端当前状态
- 顶部先展示统计总览
- 总览统计支持点击查看待办事项和已完成明细
- 工作台内各模块只展示一句简介、关键状态和操作入口
- 聊天列表中的主标题直接显示任务名称，副标题强调协作对象与当前阶段
- 从人才详情页进入聊天时，需要按目标人才上下文进入对应会话；如果同一人才存在多个任务房间，不自动误跳到旧房间
- 入驻与审核状态查看
- 发布任务
- 语音或文字输入需求
- AI 拆解需求与工期
- 对首页改版、品牌官网、H5 展示、工作台视觉升级这类首版设计/前端任务，默认按 3 到 5 个 AI 协同工作日估算，除非范围明显更大
- 手动确认拆解结果
- 查看 AI 推荐候选人才（默认 4 位）
- 从推荐名单中直接选择人才
- 不满足时进入人才广场做补充搜索与筛选
- 查看人才详情
- 和人才沟通并确认需求、工期
- 查看项目进度与验收状态

#### 企业端入驻页

- 采用单列步骤式引导
- 页面文案尽量简洁，不堆叠大段说明文字
- 先填写企业基础信息
- 再填写联系人与合作偏好
- 合作偏好优先使用结构化筛选项，不使用自由文本
- 然后确认信息摘要
- 最后进入资料上传页
- 支持当下直接上传资料
- 资料提交时需记录已选文件信息，便于后续审核查看
- 也支持先提交基础信息，后续在企业工作台补交材料
- 不论当前上传还是稍后补交，提交成功后都进入企业工作台
- 资料状态、审核状态等用户可见状态统一显示中文说明，不直接暴露内部状态码

#### 人才端工作台

- 以单列总览页展示当前接单状态
- 顶部先展示评分、收入、开放档期和进行中任务
- 工作台内各模块只展示摘要和入口，详情通过跳转或弹窗查看
- 申请入驻
- 编辑个人资料、专长、作品、评价展示
- 浏览任务广场
- 查看自动分配任务
- 任务管理
- 接单日历
- 上传进度附件
- 消息中心
- 收入与作品统计

#### 人才详情页

- 与官网首页视觉结构区分开，按简历页思路展示
- 单列布局，避免多列信息拥挤
- 展示头像、名称、专业方向、简介、履历、平台工作结果和评价
- 作品区支持图片 / 视频 / 页面链接三类内容
- 图片与视频先在轮播中浏览，再点击弹窗看大图或播放视频
- IT 类项目支持外部页面链接跳转

#### 协作空间

- 企业端协作空间支持多任务切换，每个任务都能单独查看当前协作状态
- 企业端以图形化节点或时间线展示阶段进度，节点按任务模块或工作日组织
- 点击节点可查看计划交付、人才提交、AI 审核、企业反馈和 AI 补充解释
- 企业端可以在节点详情里补充改进建议，并让 AI 对建议进行补充解释
- 企业端协作空间需要提供“发起提前完成”“申请取消任务”“进入验收与评级”的任务级入口
- 人才端协作空间保持单列执行台风格，重点承接当前任务、进度提交和附件上传
- 人才端可填写完成度、进展说明、需要协助的事项和进度附件
- 人才端协作空间需要提供“同意提前完成 / 继续执行”“同意取消 / 继续任务”的任务级处理入口
- 任务详情通过弹窗查看，不把完整任务信息塞进主区域
- AI 进度审查与修改建议统一沉淀在协作空间中
- 验收提醒与双向评分保留在协作空间闭环里

### 5.2 后台管理

- 经营看板
- 用户管理
- 企业资料审核
- 个人实名与虚拟企业审核
- 任务管理
- 风控与异常提醒
- 手工创建用户
- 手工创建任务
- 页面主视图以列表和表格为主
- 操作入口统一放在表格右侧操作栏
- 创建、审核、流转、风控处理通过右侧抽屉完成
- 仅读取必要的协作审计留痕，不承接企业端 / 人才端 IM 主会话

### 5.3 AI 能力

- 任务文本 / 语音需求拆解
- 工期评估
- 技能标签提取
- 标签优先生成任务 / 业务 / 交付标签，不默认输出技术框架标签
- 推荐内容优先选择国内可用工具、国内协作方式和国内可落地的办公 / 设计 / AI 工具
- 风险提醒
- 基于数据库筛选候选人才并输出推荐理由
- 进度完整度检查
- 验收前提醒
- 提前完成审核，重点检查当前进度、质量、附件和刷单风险
- 企业评级后的结果解释与补充建议

### 5.4 消息能力

- 平台任务会话
- 任务系统消息
- 主要服务于企业端招聘方、人才端服务方和 AI 系统助手之间的协作
- 聊天列表主标题展示任务名称，副标题展示协作对象和阶段
- 已确认任务仍保留企业端发起变更入口，聊天内任务确认卡继续显示版本与处理历史
- 企业端待人才确认的新版本支持撤回；撤回后恢复到上一版已确认内容
- 后台仅在投诉、争议、风控时查看摘要或审计留痕
- 后续接入腾讯 IM 的单聊 / 群聊 / 离线推送能力

## 6. 非功能需求

- 前台需支持 H5 与 PC 自适应
- 管理后台以 PC 宽屏为主
- 管理后台主色调采用蓝色、深蓝和白色
- 资料、身份证等敏感信息必须仅后台可见
- 核心页面结构需清晰，支持后续真实接口接入
- 代码结构需支持继续迭代

## 7. MVP 边界

本期优先实现：

- 页面骨架
- 角色流程展示
- Java mock API
- SQL 表结构
- AI 与腾讯 IM 接入预留
- 轻量账号注册 / 登录演示链路

本期暂不实现：

- Spring Boot + MySQL + JWT 正式登录注册
- 真实文件存储
- 真实支付与结算
- 真实大模型调用
- 真实腾讯 IM SDK 接入

## 8. 验收标准

### 产品层

- 能完整展示官网首页、企业端、人才端、后台管理四类页面层次
- 能演示企业端发布任务 -> AI 拆解 -> 推荐 4 位候选人才 -> 手动补充筛选 -> 协作 -> 验收流程
- 能演示人才端看任务 -> 接单 -> 协作 -> 验收 -> 评价流程
- 能体现 H5 / PC 适配能力

### 技术层

- `frontend` 可运行
- `admin` 可运行
- `backend` 可启动并返回 JSON
- SQL 表结构覆盖核心业务实体
