# 企业雇佣协作结算平台功能清单

> 版本：2026-03-30  
> 适用对象：老板、产品、设计、研发、测试、交付  
> 事实源说明：本文件是面向业务视角的功能清单，用于回答“这个项目现在有哪些功能、覆盖到什么深度、分别给谁用”。产品范围、流程边界和实现状态仍分别以需求文档、设计文档和版本状态总览为准。

## 1. 文档目标

这份清单主要解决 4 个问题：

- 给老板快速看懂：这个平台到底卖什么、做到了什么、还能扩什么
- 给产品快速盘点：当前已有模块、页面、角色、流程和能力边界
- 给设计和研发对齐：每个模块属于谁、在哪个端、当前做到哪一步
- 给测试和交付使用：知道应该按哪些功能域做检查和演示

## 2. 一句话概述

这是一个面向企业与人才的“雇佣协作结算平台”，覆盖从找人、确认合作、在线协作、验收评级到请款、开票、对账、结算、争议处理的完整链路。

它不是单纯招聘站，也不是单纯聊天工具，而是把以下能力放到一个平台里：

- 招聘撮合
- 在线沟通
- 任务协作
- 验收与评级
- 财务与结算
- 风控与争议
- 信用沉淀与复购

## 3. 适合老板先看的摘要

### 3.1 当前平台能做什么

- 企业可以注册、认证、发布任务、筛选人才、发起聊天、确认合作、推进协作、发起验收、做评级与结算
- 人才可以注册、实名认证、完善资料、浏览任务、接受邀约、确认需求、提交进度、上传附件、确认验收并沉淀接单记录
- 平台可以承接审核、审批、请款、开票、对账、结算和争议风控
- PC 和 H5 已经分端建设，PC 偏管理与多任务，H5 偏高频处理与即时协作

### 3.2 当前产品的商业价值表达

- 对企业：更快找到可交付的人，并且把项目做完、验收完、结算完
- 对人才：更清楚地接单、协作、收款、沉淀信用和复购机会
- 对平台：形成可留痕、可审核、可风控、可复用的交易闭环

### 3.3 当前功能层级

- 已具备前台主链：找人、聊天、确认、协作、验收、记录
- 已具备财务主链第一阶段：请款、开票、对账、结算
- 已具备争议与风控第一阶段：争议对象、风险工单、后台合规页承接
- 已具备 H5 与 PC 双端承接
- 当前继续精修中的方向：PC / H5 前台全页 UI 精修、Admin 全页 UI 精修
- 仍需继续强化的方向：组织权限、更细财务审计、支付出账、部分后台治理能力、上传与 IM 附件治理、容量与可观测性增强

## 4. 角色与端说明

### 4.1 角色

- 企业端：企业主、招聘负责人、项目负责人、业务协作人
- 人才端：自由职业者、外包成员、兼职人才、独立开发者 / 设计师 / 内容创作者
- 平台端：运营、审核、财务、风控、管理后台使用者

### 4.2 终端

- PC Web：适合多任务、重管理、重筛选、重协作、重列表和桌面工作区
- H5 仿 App：适合高频处理、即时聊天、待办承接、单任务执行和移动端确认
- Admin：适合审核、风控、任务管理、财务管理和后台治理

## 5. 功能总图

平台功能可以分成 8 大类：

1. 官网与账号入口
2. 企业侧找人和发单
3. 人才侧找任务和接单
4. 聊天与任务确认
5. 协作执行与过程留痕
6. 验收、评级、记录复盘
7. 财务与结算主链
8. 平台审核、风控与后台治理
