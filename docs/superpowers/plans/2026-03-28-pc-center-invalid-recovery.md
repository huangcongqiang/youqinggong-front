# 2026-03-28 PC 中心页失效态安全恢复动作

## 目标

当通知中心 / 审批中心进入“当前对象失效态”时，不只展示提示，还要提供安全恢复动作，让用户能清掉失效锚点并回到当前分组的可处理事项。

## 本阶段范围

- [ ] 通知中心补失效态恢复动作
- [ ] 审批中心补失效态恢复动作
- [ ] 恢复动作不能静默跨分组
- [ ] 清除失效锚点时保留当前 group 与中心来源
- [ ] 根 `docs/` 同步“中心页失效态恢复动作”口径

## 设计方向

- 失效态优先提供 2 类动作：
  - `查看当前分组首项`
  - `清除当前定位`
- 若当前分组已空：
  - 通知中心可提示切换全部或其它分组
  - 审批中心只提示切换分组，不自动跨组
- 恢复动作必须安全：
  - 不自动跳无关对象
  - 不偷改来源链
