# 2026-03-28 PC 列表页办公端收口

## 目标

把 PC Web 端的记录页、企业人才广场、人才任务广场继续从“大卡片流”收成更标准的办公端列表页，重点加强固定筛选、行级动作、批量处理与稳定详情承接。

## 本阶段范围

- [x] 企业 / 人才共用记录页进一步表格化，补齐批量选择与批量动作骨架
- [x] 企业人才广场改成更稳定的资源目录工作台
- [x] 人才任务广场改成更稳定的任务池工作台
- [x] 抽取一组共享的办公端列表样式与表格样式
- [x] 根 `docs/` 与四个子项目副本文档同步本阶段结果
- [x] 运行 `frontend` 与 `frontend_mobile_h5` 构建验证

## 设计方向

- 冷静办公端，而不是营销卡片流
- 顶部固定筛选，正文以行式信息为主
- 每行优先展示标题、状态、关键字段、主动作
- 批量处理先做视觉和交互骨架，不在这一阶段补完整业务

## 后续阶段

- PC 通知中心与审批入口
- 列表页批量处理的真实业务接线
- 审批、请款、结算与争议链路

## 本阶段结果

- 记录页已收成“摘要条 + sticky 工作带 + 行式列表 + 批量操作骨架”的办公端结构
- 企业人才广场已改成“左侧资源目录 + 右侧稳定详情承接”的目录工作台
- 人才任务广场已改成“固定筛选 + 左侧任务池 + 右侧详情轨”的任务池工作台
- `frontend/src/style.css` 已补共享办公端列表 / 目录样式 primitive
- `frontend` 与 `frontend_mobile_h5` 构建验证已通过
