# H5 前台收口主线

## 主线目标

把 `frontend_mobile_h5` 按 2026-03-28 的 H5 / PC 界面规范收成真正的移动端前台，而不是缩窄后的网页版本。

## 主线状态

状态：`阶段性完成`

## 进入条件

- H5 已独立成单独前端项目
- H5 聊天、工作台、协作、记录、验收仍明显带有桌面网页心智

## 完成条件

- H5 工作台、协作页、记录 / 验收详情页、聊天列表 / 详情页全部完成移动端单焦点收口
- H5 前台主路径不再依赖桌面端页面结构
- 本主线结果已回写到项目总主线计划

## 阶段拆分

### 第一阶段：工作台收口为 App 首页

目标：

- 企业端 / 人才端工作台只保留待办、两个主入口和更多信息
- 首屏不再堆叠模块卡、重复摘要和重详情
- 记录、协作、资料等次级入口统一下沉

状态：`已完成`

结果：

- 企业端工作台已收成“优先处理 + 项目聊天 + 发布任务 + 更多信息”
- 人才端工作台已收成“优先处理 + 项目聊天 + 任务广场 + 更多信息”

### 第二阶段：协作页减重为单焦点执行台

目标：

- 首屏只保留当前任务、主动作、次动作和当前节点
- 任务切换、完整节点、最近记录和重表单下沉到 sheet / 第二屏
- 去掉重复的 sticky 动作与并列入口

状态：`已完成`

### 第三阶段：记录 / 验收详情链 App 化

目标：

- 记录列表首屏去掉重摘要卡
- 记录详情首屏先给结论、动作和当前状态
- 验收页首屏只保留当前唯一待处理动作

状态：`已完成`

结果：

- H5 记录列表已保持为轻量列表首屏，不再回到摘要卡堆叠结构
- H5 记录详情首屏当前统一为“结论 + 动作 + 当前状态 + 附件留痕”
- H5 验收页首屏当前统一为“当前结论 + 当前唯一待处理动作 + 动作结果”
- 过程节点、确认历史、信用影响、评分历史与评级依据已统一下沉到折叠层

### 第四阶段：对象页单焦点收口

目标：

- H5 聊天详情、记录详情、验收页进一步统一成“一个当前对象 + 一个主动作 + 少量首屏状态”
- 继续减少首屏并排摘要和次级说明
- 重历史、重留痕、重说明优先下沉到折叠层或 sheet

状态：`已完成`

结果：

- H5 聊天详情页已移除重复的 `SummarySection`，当前首屏优先只保留会话头、任务确认卡、消息线程和输入区
- H5 任务确认卡已压缩为预算、工期、协作安排、档期和主动作，版本历史与更长说明已下沉到任务详情 / 任务确认 sheet
- H5 记录详情首屏已继续压缩状态区，更接近对象摘要，而不是 dashboard 式对象页

### 第五阶段：聊天数据层与实时链瘦身

目标：

- 继续拆 H5 聊天详情页里的 IM 连接、房间同步、消息发送和轮询逻辑
- 让 `MessagesPage` 更像对象页，而不是页面里混着一整套数据层
- 在不改业务链路的前提下继续压缩聊天页 chunk 和页面复杂度

状态：`已完成`

结果：

- H5 聊天详情页已把消息展示、附件展示、任务确认历史整理、消息合并和状态映射抽到 `messageDetailHelpers.js`
- H5 聊天详情页已把轮询刷新开关与刷新间隔判断抽到 `messageLiveRefresh.js`
- H5 聊天详情页已把房间预选、对象级空态房间、房间摘要 enrich 与详情刷新判断抽到 `messageRoomRuntimeHelpers.js`
- `MessagesPage.vue` 当前更接近“页面编排层 + 视图层”，后续若继续拆 `useMessageRoomSession` 会更顺
