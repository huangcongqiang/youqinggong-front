## 设计基线

1. 列表优先。
2. 抽屉或右侧操作栏承接编辑/审核。
3. 不做前台式营销卡片堆叠。
4. 收敛到“左导航 + 顶部状态 + 主工作区”。
5. 审核、风控、任务流转、财务治理优先使用表格/列表 + 行级动作。
6. 后台不承接 IM、聊天、协作、验收等前台主操作面板。

## 审查重点

1. `Auth` 必审登录态与创建测试账号态。
2. `Dashboard` 单独按看板页审，不退回展示型首页。
3. `Users` 重点看用户列表、审核队列、用户详情与审核处理抽屉。
4. `Tasks` 重点看任务列表、流转队列、任务详情与流转处理抽屉。
5. `Compliance` 重点看风险工单、合规清单、交付开发板、组织权限与统一处理抽屉。
