# 2026-03-28 PC 记录详情与聊天右侧上下文

## 目标

把 PC Web 的记录详情页与聊天页右侧上下文收成同一套对象语义：从记录详情进入聊天时，聊天页右侧能继续看到这条记录对应的任务结论、时间线摘要和关键结果；从聊天返回记录详情时，也能保留同一条记录的上下文。

## 本阶段范围

- [x] 梳理记录详情与聊天右侧上下文的共享字段与入口链路
- [x] 聊天页右侧补记录级摘要与记录详情入口
- [x] 记录详情进入聊天、返回记录详情时保持 `recordId / taskId / room / source / tab`
- [x] 根 `docs/` 与四个子项目副本文档同步本阶段结果
- [x] 运行 `frontend`、`frontend_mobile_h5`、`backend` 验证

## 设计方向

- 记录详情与聊天不是两套割裂对象，而是同一任务的“复盘面”和“沟通面”
- 聊天右侧上下文优先承接记录级结论、状态、金额、评级和快捷返回
- 记录详情进入聊天后，返回链优先回到原记录详情，而不是回到记录列表或最近聊天

## 后续阶段

- 批量通知 / 审批处理与聊天详情联动
- 审批中心与协作空间的状态级联更新
- PC 记录页批量动作正式化

## 本阶段结果

- 记录详情进入聊天后，聊天右侧已不再只有通用任务摘要，而是会继续展示当前记录的阶段、金额、评级、最近留痕和查看记录入口
- `recordId / taskId / room / source / tab` 这条链在“记录详情 -> 聊天 -> 返回记录详情”路径上继续保留，避免对象语义丢失
- 聊天右侧当前只承接轻量记录对象事实，不直接搬完整时间线或完整资产列表，保证右栏仍然适合决策
