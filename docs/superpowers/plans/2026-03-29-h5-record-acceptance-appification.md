# H5 记录 / 验收详情链 App 化

## 阶段目标

- 记录列表首屏保持轻量，只先承接标题、tab、数量和列表
- 记录详情首屏只保留结论、动作、当前状态和附件直达
- 验收页首屏只保留当前结论、当前唯一动作和结果摘要
- 过程节点、确认历史、信用影响、评分历史等追溯信息统一下沉

## 实际结果

- `RecordPage.vue` 保持轻量列表页，不再回到摘要卡堆叠结构
- `RecordDetailPage.vue` 现在首屏固定为“结论 + 动作 + 当前状态 + 附件留痕”，补充信息、交付范围、最新留痕、确认历史和过程节点已统一折叠到 `DetailAccordion`
- `AcceptancePage.vue` 现在首屏固定为“当前结论 + 当前唯一待处理动作 + 动作结果”，评级依据、评分反馈、信用影响和过程节点已统一折叠到 `DetailAccordion`
- H5 记录 / 验收详情链当前已经更接近 App 对象页，而不是缩窄后的 PC 详情页

## 验证

- `frontend_mobile_h5`：`npm run build` 通过
