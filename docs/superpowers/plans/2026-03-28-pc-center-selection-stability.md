# 2026-03-28 PC 中心页当前对象稳定态与失效态

## 目标

让通知中心 / 审批中心在轮询刷新、切组或当前事项消失时，不再静默漂到别的事项上；当前对象命中失败时，应明确进入“已失效 / 待重选”状态。

## 本阶段范围

- [x] 通知中心补当前对象失效态
- [x] 审批中心补当前对象失效态
- [x] 当 URL 显式带 `itemId / taskId / room / recordId` 且命中失败时，不回退到无关首项
- [x] 用户手动重新选事项后，失效态应被正确清掉
- [x] 根 `docs/` 同步“中心页当前对象稳定态”口径

## 本阶段结果

- 通知中心与审批中心当前都新增了 `selectedItemMissing` 失效态，用于区分“当前对象失效”和“当前列表为空”
- 当 URL 显式带 `itemId / taskId / room / recordId` 且当前事项命中失败时，页面当前会保留原分组语义并进入失效态，不再静默切到无关首项
- 用户手动切组或手动重新选择事项后，失效态会被清掉，页面重新进入稳定态
- 顶部摘要高亮、主动作按钮和中栏说明当前都会跟随稳定态 / 失效态切换，不再出现右侧失效、上方仍像命中成功的状态分裂

## 设计方向

- 有明确对象锚点时：
  - 命中：锁定当前对象
  - 失效：显示“当前事项已处理或不可用”，不自动换成别的事项
- 无明确对象锚点时：
  - 允许退回到当前分组第一条，保持中心页可用
- 右侧摘要区与动作带要和稳定态一致：
  - 有当前对象才显示对象摘要与动作带
  - 失效态时显示重选提示，不伪装成别的对象
