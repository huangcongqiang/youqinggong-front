# 项目连续推进规则

## 背景

从 2026-03-29 开始，项目内任何采用“主线 -> 阶段”方式推进的工作，默认都使用连续推进模式执行，而不是在单个阶段完成后停住等待再次确认。

这份规则的目标已经从“阶段连续推进”升级为“项目连续推进”，直到主线 10、11、12 全部完成为止。

## 全项目默认规则

1. 默认使用 `superpowers` 工作流推进当前项目、当前主线和当前阶段。
2. 当前线程如已具备完全访问、且本地执行无需审批的条件，后续所有本地文件、端口、截图、`dev server`、文档同步、证据归档等动作默认直接执行，不再重复确认权限。
3. 默认尽量使用 `5` 个 agent；如果线程上限、任务依赖或资源约束不允许，再降级，但仍要保持高并发和连续推进。
4. 每完成一个阶段，都必须先更新该阶段计划文档，再回写主线总计划文档，然后立即切到下一阶段；如果当前主线已经完成，则立即切到下一主线。
5. 每完成一条主线，都必须先回写主线总计划文档，再立即切入项目总计划中的下一条主线，直到主线 10、11、12 全部完成。
6. 对于仍需继续推进的阶段，必须再拆出“切片级执行计划”，明确当前活跃切片、下一切片和切片收口条件。
7. 每完成一个阶段，都必须在必要时回写开发文档、版本状态总览、测试清单与技术文档。
8. 截图必须按需获取，不做分批量预抓；`/tmp` 仅作为采集临时区，正式证据必须立即归档到 `docs/superpowers/artifacts`。
9. 如果当前阶段已拆成切片，则每个切片都必须先完成根 `docs/` 回写、四个 `docs` 副本同步、整端 UI 审查、全量主路径功能回归和阻断项清零，才允许把下一切片标记为 `进行中`。
10. 只有主线 10、11、12 全部完成，才允许把整个项目标记为完全完成。

## 当前默认执行参数

1. 并行阶段默认尽量使用 `5` 个 agent。
2. 如果受线程上限限制，则优先复用现有 agent，并保持高并发推进。
3. 阶段完成后默认继续，不在单阶段完成点停住。
4. 主线完成后默认继续，不在单条主线完成点停住。
5. 项目未满足“项目完成定义”前，默认不允许把推进动作停留在“待决定”“后续再看”或“等确认”状态。

## 适用对象

以下工作默认适用本规则：

- H5 前台收口主线
- PC Web 前台收口主线
- 文档与实现对齐主线
- PC / H5 共享数据层与运行时收口主线
- `spring-app` 正式主链接管主线
- 审批 / 结算 / 争议等正式业务主链
- 主线 10：PC + H5 前台全页 UI 截图评审与改版
- 主线 11：Admin 全页 UI 截图评审与改版
- 主线 12：上线后增强项收口

## 主线文档必须具备的字段

每条主线至少需要明确：

1. 主线目标
2. 主线范围与不在本轮范围
3. 阶段列表与状态
4. 当前活跃阶段
5. 下一阶段默认入口
6. 完成条件 / 阶段性完成条件
7. 风险与阻塞
8. 阶段索引或对应阶段文档链接
9. 唯一当前活跃阶段

如果某一阶段本身跨度较大，还必须明确：

10. 当前活跃切片
11. 下一切片默认入口
12. 切片级最小验证
13. 切片级 UI 审查范围
14. 切片级全量主路径回归范围

## 项目完成定义

整个项目只有在以下条件全部满足时，才允许标记为“整个项目完全完成”：

1. 主线 10 已完成，并且对应阶段、切片、截图证据和文档回写都已收口。
2. 主线 11 已完成，并且对应阶段、切片、截图证据和文档回写都已收口。
3. 主线 12 已完成，并且对应阶段、切片、证据和文档回写都已收口。
4. 项目总计划中不再存在任何“进行中”的主线、阶段或切片。
5. 正式证据已从 `/tmp` 采集区归档到 `docs/superpowers/artifacts`。
6. 当前活跃主线、当前活跃阶段与当前活跃切片都已清零，并且总计划状态已明确写成“项目完全完成”。

只要主线 10、11、12 里还有任何一条未完成，就只能叫“项目继续推进中”，不能宣布整个项目完成。

## 阶段收口最低要求

每阶段结束前至少完成以下动作：

1. 明确阶段是否完成、剩余风险和下一阶段入口。
2. 如果阶段未完成，必须把当前活跃切片、下一切片和切片状态一起回写。
3. 同步根目录 `docs/` 与四个子项目 `docs` 副本。
4. 运行与本阶段改动相匹配的最小验证命令。
5. 将下一阶段作为默认继续动作，而不是留在“待决定”状态。
6. 若阶段已拆成切片，则必须显式记录当前切片的整端 UI 审查结果与全量主路径回归结果。
7. 若阶段产出截图证据，必须先放入 `/tmp` 采集区，再立即归档到 `docs/superpowers/artifacts`。

## 阶段完成后的标准输出

每阶段至少要留下这 5 类结果：

1. 阶段结果
2. 风险与已知缺口
3. 验证结果
4. 文档同步结果
5. 下一阶段或下一条主线

补充硬要求：

- 每个阶段计划文档的最后一项，必须明确写成“继续下一阶段”或“切到下一条主线”。
- 任一主线在任一时刻只能有一个“进行中”的当前活跃阶段；阶段收口后必须立即把下一阶段改为“进行中”。
- 任一跨度较大的阶段在任一时刻也只能有一个“进行中”的当前活跃切片；切片收口后必须立即把下一切片改为“进行中”。
- 不允许把阶段文档停在“待决定”“后续再看”“如有需要再继续”这种中间态。
- 当前主线完成后，必须立即切到项目总计划中的下一条主线，直到主线 10、11、12 全部完成。

## 例外情况

只有在以下情况出现时，才允许中断连续推进：

- 当前阶段遇到无法在本地自主解决的真实阻塞
- 继续推进会产生高风险的破坏性后果
- 用户明确要求暂停、转向或结束当前主线
- 当前主线已经完成，并已在项目总计划中显式切到下一条主线
