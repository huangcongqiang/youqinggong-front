# `审批 / 请款 / 开票 / 对账 / 结算 / 争议主链` 主线

## 主线目标

把当前“审批中心已有入口、财务链和争议链尚未冻结”的状态，推进到：

- 审批对象有统一状态机与正式写链
- 请款、开票、对账、结算进入可审计对象主链
- 争议与风控升级进入正式后台/前台联动链

## 范围

- 审批对象与状态机
- 请款 / 开票 / 对账 / 结算对象与写链
- 争议对象与风控升级
- 前台中心页、记录详情、聊天右侧上下文对这些对象的最小消费合同

## 当前活跃阶段

- 当前进行中：无

## 已完成阶段

- 阶段一：`审批 / 财务状态冻结`
- 阶段二：`审批中心正式化`
- 阶段三：`请款最小闭环`
  - `spring-app` 已新增 `Claim` 正式对象与最小创建 / 审批动作接口
  - 记录详情当前已消费 `claimSummary`
  - 任务生命周期当前已写入 `CLAIM` 审计事件
- 阶段四：`开票与对账`
  - `spring-app` 已新增 `Invoice / Reconciliation` 正式对象与最小创建 / 对账动作接口
  - 记录详情当前已消费 `invoiceSummary / reconciliationSummary`
  - 财务摘要当前已按同一条 `claim -> invoice -> reconciliation` 链聚合
  - 已补“提交开票后、待企业对账”的中间态订单详情回读测试
- 阶段五：`结算执行`
  - `spring-app` 已新增 `Settlement` 正式对象与最小执行动作接口
  - 对账确认后当前会自动生成 `PENDING_EXECUTION` settlement，并补 `SETTLEMENT` 生命周期事件
  - 记录详情、闭环详情当前都已消费 `settlementSummary`
  - 财务摘要当前已按同一条 `claim -> invoice -> reconciliation -> settlement` 链聚合
- 阶段六：`争议与风控升级`
  - `spring-app` 已新增正式 `Dispute` 对象与最小状态流转
  - 企业对账提出异议后，当前会创建或更新 `Dispute` 与 `RiskTicket`
  - 记录详情、闭环详情当前都已消费 `disputeSummary`
  - 后台合规页当前可回读 `targetType=DISPUTE` 的风险工单，并可通过更新 risk ticket 状态回写争议状态
