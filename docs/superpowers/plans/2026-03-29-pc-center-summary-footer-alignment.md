# 2026-03-29 PC 中心页摘要卡与 Footer 动作对齐

## 目标

把通知中心 / 审批中心在 `stable / invalid` 两种对象状态下的顶部摘要卡、右侧空态与底部 footer 动作统一到同一套语义，不再出现顶部、右侧、底部各说各话。

## 本阶段范围

- [x] 通知中心顶部摘要卡在失效态切到恢复动作
- [x] 审批中心顶部摘要卡在失效态切到恢复动作
- [x] 通知中心 footer 在失效态复用同一组恢复动作
- [x] 审批中心 footer 在失效态复用同一组恢复动作
- [x] 右侧空态与顶部 / 底部动作保持一致
- [x] 根 `docs/` 同步“中心页摘要卡与 footer 动作对齐”口径

## 设计方向

- `stable` 态继续保留对象动作与跨中心动作
- `invalid` 态不再保留对象级 CTA
- `invalid` 态优先暴露：
  - `查看当前分组首项`
  - `查看全部事项`
  - `清除当前定位`
- 顶部摘要卡、右侧空态、底部 footer 都消费同一套恢复动作，避免状态漂移

## 本阶段结果

- 通知中心当前已新增 `invalidRecoveryActions`，顶部摘要卡、右侧空态与 footer 在对象失效时统一切到恢复模式
- 审批中心当前也已新增同样的恢复动作合同；顶部摘要卡 secondary、footer 和右侧空态不再继续保留误导性的常规对象动作
- 审批中心在对象失效但当前分组仍有事项时，当前会优先定位“当前分组首项”；若当前分组为空，则改成“清除当前定位 + 查看全部通知”
- 审批中心与通知中心的恢复动作当前都只在中心页内生效，不会把用户直接带去无关对象页
