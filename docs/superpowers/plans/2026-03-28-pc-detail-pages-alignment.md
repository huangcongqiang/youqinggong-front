# PC 详情页办公端化对齐进展

## 阶段目标

- 把 PC Web 的记录详情页、人才详情页继续从展示型详情收成更像办公端的对象详情页。
- 保持与 `2026-03-28-h5-pc-ui-spec.md` 和 `2026-03-28-enterprise-employment-collaboration-design.md` 中“PC 偏详情工作区、信息更集中、上下文更稳定”的方向一致。
- 让列表页和详情页形成统一的“左侧列表 / 详情入口 / 详情承接”心智。

## 当前阶段拆分

- [x] 记录详情页收成办公端详情结构，突出概览、关键结果、时间线与入口动作。
- [x] 人才详情页收成办公端对象详情结构，突出能力、作品、合作方式和触发动作。
- [x] 整理共享详情页样式，统一为 PC 详情页视觉语言。
- [x] 同步根文档与四个子项目 docs 副本。
- [x] 跑 PC 与 H5 构建验证，记录下一阶段建议。

## 并行策略

- 子线程 A：记录详情页结构改造
- 子线程 B：人才详情页结构改造
- 子线程 C：详情页样式建议
- 子线程 D：文档口径与设计对齐检查
- 子线程 E：当前详情页与新列表页之间的衔接审查
- 主线程：整合、样式落地、文档同步、构建验证

## 本阶段结果

- PC 记录详情页已改为更偏办公端复盘页的结构：第一屏结论、关键结果、时间线、资产与主动作入口。
- PC 人才详情页已改为更偏对象决策页的结构：结论、关键能力、作品、合作方式与平台表现分区展示。
- 已补共享 `office-detail-*` 壳样式，用于统一 PC 详情页的顶部结论区、两栏布局与右侧动作区。
- 已补路由连续性：企业端人才广场筛选会带入人才详情并在返回时恢复；记录详情进入聊天会同步带上 `taskId / recordId`。
- 已同步根 `docs/` 与 `frontend / frontend_mobile_h5 / admin / backend` 的 `docs` 副本。
- 已验证：
  - `frontend` `npm run build`
  - `frontend_mobile_h5` `npm run build`

## 下一阶段建议

- 继续推进 PC 列表页与详情页之间的“上下文不丢失”能力，例如列表滚动位置、选中对象和更多筛选项。
- 在 PC 协作空间继续补强“左任务树 / 中主区 / 右详情”里的任务树深度、筛选和批量动作。
