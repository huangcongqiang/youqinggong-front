# 2026-03-28 PC 中心页当前对象动作带

## 目标

给 PC 通知中心 / 审批中心补一条“当前对象动作带”，让用户在中心页完成当前事项预选后，可以直接从当前对象进入聊天、协作空间、记录详情、验收页，而不是继续依赖列表行内零散入口或右侧大段说明区。

## 本阶段范围

- [x] 通知中心补“当前对象动作带”
- [x] 审批中心补“当前对象动作带”
- [x] 动作带统一消费当前选中对象，而不是重复做一套列表行动作
- [x] 动作带进入聊天 / 协作空间 / 记录详情 / 验收页时继续保留 `group + itemId + taskId / room / recordId / source / originSource`
- [x] 缺少目标链路的动作只展示禁用态或隐藏，不做静默 fallback
- [x] 根 `docs/` 同步“中心页当前对象动作带”的阶段口径

## 本阶段结果

- 通知中心右侧“当前摘要”已改成围绕 `selectedItem` 的对象动作带，支持按可达性显示 `去聊天 / 去协作空间 / 看记录详情 / 去验收页 / secondaryRoute`
- 审批中心右侧“当前决策摘要”已补对象动作带，支持按可达性显示 `去处理 / 去聊天 / 去协作空间 / 看记录详情 / 去验收页 / 去通知中心`
- 两个中心页的顶部摘要高亮当前都优先跟随当前选中对象，不再只按列表前几项漂移
- 动作带继续使用当前对象锚点 `group + itemId + taskId / room / recordId`，并保留既有 `source + originSource` 回链规则

## 设计方向

- 动作带属于“中心页当前对象”的固定工作带，不是新的说明卡，也不是列表行内的第二套按钮区
- 摆放优先级：
  - 放在右侧当前对象摘要区顶部，或当前对象标题下方
  - 与当前对象标题、状态、轻摘要同层
  - 不额外增加大块说明文案
- 动作带只保留对象工作页动作，不承接中心页级动作：
  - `去聊天`
  - `去协作空间`
  - `查看记录详情`
  - `去验收页`
- 企业端审批中心与人才端通知中心共用同一动作带语义，但允许按对象可达性裁剪动作数量
- 动作带的本质是“当前对象二次分发”，不是“又一层首页导航”

## 路由与上下文约定

- 动作带永远以“当前选中对象”为源，不自行重新推断对象
- 进入对象页时继续带上：
  - `group + itemId`
  - `taskId / room / recordId`
  - `source=notifications|approvals`
  - `originSource=notifications|approvals`
- 返回链保持现有规则：
  - 对象页返回中心页时优先恢复 `itemId`
  - 仅在 `itemId` 失效时，再按 `recordId -> taskId -> group`
- 当用户在中心页切换分组或切换当前事项后，动作带必须立即切换到新对象，不允许粘住旧对象锚点

## 交互原则

- 动作带必须轻量，优先像桌面办公端的对象工具条，而不是营销式 CTA 区
- 不新增“查看全部上下文”“展开更多说明”这类块级文案
- 不把审批中心 / 通知中心之间的互跳塞进当前对象动作带
- 不把批量操作塞进当前对象动作带；批量动作仍属于列表级工具条
- 行内列表入口可以保留最少量直达，但“当前对象动作带”应成为中心页最稳定的对象动作入口

## 验证矩阵

- 通知中心选中某事项后，动作带能稳定显示该事项可达的聊天 / 协作空间 / 记录详情 / 验收页动作
- 审批中心切换当前事项后，动作带不会继续保留旧事项的 `itemId / taskId / room / recordId`
- 从动作带进入聊天、协作空间、记录详情、验收页后，返回仍能恢复原中心页分组与原事项
- 某事项缺少 `room`、`recordId` 或验收入口时，动作带只做禁用或隐藏，不静默跳到默认对象

## 预期结果

- 中心页的“当前对象”会真正形成桌面工作焦点，而不是只靠列表高亮和右侧摘要
- 用户从中心页进入对象页的路径更稳定，首页入口、中心页预选、对象页返回三层职责更清晰
- 后续如果要继续补审批操作、已读状态或跨中心互跳，仍可以围绕当前对象动作带继续收口，而不用再把入口撒回列表行里
