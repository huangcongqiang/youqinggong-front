# 2026-03-28 PC 聊天来源链与上下文保持

## 目标

把 PC Web 聊天页补成真正的工作区详情页，承接来自工作台、通知中心、审批中心、协作空间和记录详情的精确上下文，避免进入聊天后丢失来源与当前任务语境。

## 本阶段范围

- [x] 聊天页补来源标识与返回来源入口
- [x] 从工作台 / 通知中心 / 审批中心 / 记录详情进入聊天时保留 `taskId / room / recordId / source`
- [x] 聊天页当前任务摘要、当前房间与来源提示对齐精确上下文

## 设计方向

- 聊天页是工作区详情，不是纯消息黑洞
- 进入聊天后，用户需要清楚知道“我从哪来、当前在处理哪个任务、可以返回哪里”
- 来源提示要轻量，不额外制造长说明区
- 返回链优先回原来源，不强制回聊天列表

## 后续阶段

- PC 协作空间与聊天的双向锚点跳转
