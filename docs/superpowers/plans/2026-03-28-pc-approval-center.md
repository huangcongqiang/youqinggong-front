# 2026-03-28 PC 企业端审批中心骨架

## 目标

把企业端当前分散在工作台顶部待办和通知中心里的“审批式动作”继续收口成独立审批中心页，承接待确认、待修改、待评级、待取消和发布后待选人等企业端高优先级事项，让 PC Web 更接近正式办公端的使用方式。

- [x] 企业端桌面壳补审批中心入口与路由
- [x] 企业工作台顶部待办主动作改为进入审批中心
- [x] 通知中心为企业端补“去审批中心”联动入口
- [x] 根 `docs/` 与四个子项目副本文档同步本阶段结果
- [x] 运行 `frontend`、`frontend_mobile_h5`、`backend` 验证

## 设计方向

- 审批中心只承接企业端高优先级决策，不混入普通聊天回看
- 页面优先展示：审批摘要、分组切换、当前审批队列、右侧详情
- 通知中心保留“全量通知”定位，审批中心强调“需要我立即决策的事项”
- 企业端入口优先从工作台顶部待办和桌面壳主导航进入
