# adr 目录说明

这个目录回答：为什么做出这些关键架构决定。

适合谁看：

- 技术负责人
- 后端开发
- 新接手项目的工程师

建议先看：

1. [ADR-0001：采用模块化单体作为后端起步架构](./0001-adopt-modular-monolith.md)
2. [ADR-0002：前台与后台管理采用双 Vue 应用](./0002-split-web-and-admin-frontends.md)

这个目录里应该放什么：

- 架构决策背景
- 决策内容
- 取舍和影响
- 为什么不用其他方案

这个目录里不应该放什么：

- 产品需求变更记录
- 接口字段明细
- 运维执行步骤
