# ADR-0002: 前台与后台管理采用双 Vue 应用

## Status
Accepted

## Context

前台应用需要同时兼容 H5 与 PC，强调品牌展示、任务协作和角色工作台；后台管理强调宽屏信息密度、审核效率和经营视角。两者在信息架构、视觉节奏和交互模式上差异明显。

## Decision

将前台与后台拆成两个独立 Vue 应用：

- `frontend`：负责官网首页、企业端、人才端、协作空间
- `admin`：负责平台管理、审核、风控、任务与用户管理

## Consequences

### Positive

- 能分别优化移动端与桌面端体验
- 构建、部署、权限和菜单结构更清晰
- 视觉风格可分别服务业务目标

### Negative

- 需要维护两套构建配置
- 共用组件需后续再抽取

### Neutral

- 当前阶段共用的数据模型和 API 契约不受影响
