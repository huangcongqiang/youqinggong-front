# ADR-0001: 采用模块化单体作为后端起步架构

## Status
Accepted

## Context

AI 人才协作市场在 MVP 阶段需要同时覆盖用户入驻、企业审核、任务发布、AI 拆单、人才匹配、协作沟通、进度反馈、验收评分和后台管理。业务链路长、实体关系强、状态变更多，但短期目标是尽快落地可演示系统，而不是优先构建复杂分布式架构。

## Decision

后端采用 Java 模块化单体架构，按领域划分为：

- onboarding
- tasks
- ai
- chat
- reviews
- admin

先保持统一部署和统一数据源，后续再按压力与团队规模拆分。

## Consequences

### Positive

- 开发速度快，联调成本低
- 数据一致性更容易保证
- 更适合当前多流程闭环业务
- 后续拆分服务仍有清晰领域边界

### Negative

- 单应用体积会逐步增长
- 后期需要补充模块隔离和服务拆分治理

### Neutral

- 需要在代码结构层面先做好领域边界管理

## Alternatives Considered

### 纯前端原型

被拒绝，因为无法验证真实接口和数据模型闭环。

### 一开始拆微服务

被拒绝，因为当前阶段运维与协作成本过高。
