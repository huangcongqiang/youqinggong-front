# 企业雇佣协作结算全流程设计文档

> 版本：2026-03-28  
> 事实源说明：本文件基于“规范文件夹/02-设计文档”整理，用于承接产品流程、信息架构和页面职责。

## 1. 设计目标

平台不是单纯招聘站，也不是单纯 IM 或项目管理工具。设计要把以下链路做成一个连续体验：

- 招人
- 确认合作
- 项目协作
- 验收结算
- 信用沉淀与复购

设计重点：

- 角色路径清晰
- 状态可追踪
- 高频路径优先
- 低频说明隐藏
- H5 与 PC 明确分工

## 2. 总体信息架构

### 前台

- 平台介绍页
- 登录 / 注册
- 企业入驻
- 人才入驻
- 企业工作台
- 人才工作台
- 人才广场
- 任务广场
- 聊天
- 协作工作区
- 验收与结算
- 发单记录 / 接单记录
- 我的页面

### 后台

- 审核台
- 风控台
- 用户与任务管理
- 结算与发票台

## 3. 企业端流程设计

1. 注册与认证
2. 发布职位/任务
3. AI/规则拆解
4. 候选人才推荐与筛选
5. 聊天与任务确认
6. 协作执行
7. 变更、审批、请款
8. 提前完成 / 取消 / 验收
9. 结算、付款、发票
10. 评分与复购

关键原则：

- 任务不是一发布就结束，而是协作链的起点
- 聊天只负责沟通，不承载完整协作工作区
- 协作页只承接执行与状态，不替代聊天

## 4. 人才端流程设计

1. 注册与实名
2. 完善资料与作品
3. 浏览任务或接受邀约
4. 聊天确认任务
5. 提交进度与附件
6. 确认验收与结算
7. 对企业评分
8. 沉淀接单记录与信用

关键原则：

- 人才先看是否值得接，再进入沟通
- 确认任务前必须能看到预算、工期、协作安排和交付要求
- 接单后必须能回看完整过程

## 5. 页面职责

### 首页 / 平台介绍页

- 只负责品牌表达、角色入口、案例与结果证明
- 不承担登录后主工作区职责
- 不在登录后继续作为默认首页

### 工作台

- 登录后默认进入
- 只显示当前最重要的待办、提醒和入口
- 不堆长说明和完整详情
- 只做“摘要 + 下一步”
- H5 工作台首屏优先保留待办列表与 2 到 3 个主入口
- PC 工作台首屏优先保留待办、风险、桌面摘要和主工作入口

### 列表页

- 人才广场 / 任务广场 / 记录列表 / 审批列表
- 只展示首屏关键字段
- 详细信息进入详情页或弹层
- PC 过渡态允许先以“摘要条 + 筛选区 + 行式列表/资源目录 + 固定详情承接”落地，避免继续沿用展示化大卡片流
- 人才广场在 PC 上优先作为候选资源池
- 任务广场在 PC 上优先作为任务池工作台
- 发单记录 / 接单记录在 PC 上优先作为全过程留痕与复盘入口

### 详情页

- 单对象、单焦点
- 第一屏先给结论
- 再给关键数据、操作入口和历史记录
- H5 详情页保持全屏承接单对象，不回退到多栏
- PC 详情页优先收成“顶部结论区 + 中部主内容 + 右侧动作/上下文”，人才详情偏对象决策页，记录详情偏全过程复盘页
- 记录详情与聊天应共享同一条记录对象语义：记录详情负责全量复盘，聊天右侧只承接轻量摘要与关键动作

### 聊天页

- 只处理当前会话
- 沟通相关卡片以消息流方式插入
- 顶部承接会话信息与更多动作
- H5 保持“会话列表页 -> 单会话详情页”
- PC 保持“左会话 / 中消息 / 右上下文”三栏工作区
- 当聊天来源带 `recordId` 时，右侧上下文优先承接记录级摘要、关键结论和返回记录详情入口

### 协作工作区

- 承接执行、进度、节点、审批、附件和验收
- PC 偏多任务与多栏
- H5 偏单任务与轻协作
- H5 首屏优先承接当前任务、关键节点与主动作，次级历史通过 sheet 或详情承接
- PC 首屏优先承接任务树或任务切换器、主工作区与固定上下文详情，不再用长单列卡片流代替
- 企业端 PC 协作空间优先承接任务动作摘要、交付节点、节点反馈与固定上下文
- 人才端 PC 协作空间沿用同一工作区壳，但主区优先执行表单与节点，次级历史下沉到侧栏或次级区

### 记录页

- 企业端叫发单记录
- 人才端叫接单记录
- 都以列表承接从确认到结束的全过程

## 6. 关键状态设计

需要统一暴露的状态：

- 待确认
- 待企业修改
- 已修改
- 已确认
- 进行中
- 待验收
- 已完成
- 已取消
- 待付款
- 已付款

要求：

- 状态必须有清晰中文
- 状态变更必须可回看
- 版本与修改记录不能丢

## 7. 重点业务节点

### 任务确认

- 企业发起确认单
- 人才确认或提修改
- 企业修改并重新发送
- 系统保留版本与处理记录

### 提前完成

- 企业发起
- AI 先审核
- 人才确认
- 企业评级

### 取消任务

- 任一方发起
- 对方同意后生效

### 评分与结算

- 企业给人才 `S / A / B`
- 人才给企业打分
- 评分回写到记录、工作台与案例结果摘要
