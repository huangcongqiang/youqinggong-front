# design 目录说明

这个目录回答：产品页面、角色入口和流程怎么设计。

适合谁看：

- 产品经理
- UI / UX
- 前端开发
- 需要确认业务流程的后端开发

建议先看：

1. [全流程设计文档](./2026-03-28-enterprise-employment-collaboration-design.md)
2. [H5 与 PC 界面规范](./2026-03-28-h5-pc-ui-spec.md)

这个目录里应该放什么：

- 页面层级和信息架构
- 企业端和人才端完整业务流程
- H5 / PC 页面范式与端边界
- 任务流、协作流、验收流与结算流
- 后台管理交互原则

这个目录里不应该放什么：

- 数据库表结构
- 接口字段细节
- 服务器部署内容
